Application Test Instructions for Non-Technical Users

Purpose:
This document explains how to test the application using common user scenarios. It is written for people who are not developers.

Preparation:
1. Open the application in your browser using the URL provided by the team (for example, http://localhost:4200 or the shared staging URL).
2. Use the test login credentials provided by the project team. If multiple roles are available, test each role separately:
   - Investor user
   - Guarantor user
   - Governor user
   - PME user (if available)
3. If the application uses wallet login or MetaMask, connect the wallet when requested and allow the signing prompts.

General Notes:
- Follow each scenario step by step.
- For fields marked with examples, enter the example text or use the provided test values.
- If the app shows an error or unexpected result, write down the exact message.
- Use the browser Back or navigation menu to move between pages.

Scenario 1: Investor Dashboard
1. Log in as an Investor user.
2. Navigate to /investor/dashboard or click "Investor dashboard" from the menu.
3. Verify the top cards display:
   - Total invested
   - Active loans
   - Average return rate
4. Check the portfolio chart and label list below it. Confirm the chart status names show FUNDED, REPAID, DEFAULTED.
5. Review the "Recent activity" list and verify each row shows:
   - PME name
   - Amount
   - Date
   - Status badge
6. Note whether the values and statuses appear correctly.

Scenario 2: Investor Marketplace Search and Fund Loan
1. Go to /investor/marketplace.
2. Confirm the page title is "Loan marketplace" and there is a search box with a search icon.
3. In the search field, type a PME name or partial text and verify the loan cards update.
4. Click each filter chip at least once: All, Low risk, High yield, Short term.
5. If loan cards appear, open one by clicking "Fund this loan".
6. In the dialog, verify the summary shows:
   - PME name
   - Amount
   - Collateral
   - Duration
   - LTV ratio
7. Click "Confirm & send ETH" and follow any wallet prompts.
8. Confirm the card updates to status FUNDED after the transaction completes.
9. If no loan cards exist, verify the empty state message appears: "No loan requests available right now." 
10. Wait 30 seconds and confirm the list refreshes automatically.

Scenario 3: Investor Investments
1. Navigate to /investor/investments.
2. Confirm the table columns include:
   - PME
   - Amount funded
   - Collateral
   - Status
   - Funded date
   - Repayment progress
3. Click column headers to sort the table.
4. If the table has multiple pages, use the paginator controls to navigate.
5. Expand a row (if available) and verify it shows:
   - Transaction hash
   - Collateral token ID
   - Blockchain link icon

Scenario 4: Guarantor Dashboard
1. Log in as a Guarantor user.
2. Go to /guarantor/dashboard.
3. Confirm the cards show:
   - Total assets provided
   - Assets in use as collateral
   - Total value guaranteed
4. If pending guarantees are expiring within 7 days, verify the amber alert banner appears with text:
   "X guarantee(s) expiring soon — review them."
5. In the "My guarantees" table, verify the last 5 entries show:
   - Asset
   - Linked PME
   - Status badge
   - Value
6. Click "View all" to go to the full guarantees page.

Scenario 5: Guarantor Guarantees Page
1. Go to /guarantor/guarantees.
2. Confirm the page header says "My guarantees" and there is an "Add asset" button.
3. Click "Add asset" and fill in the dialog fields:
   - Asset name: "Test Asset"
   - Type: choose Equipment / Real estate / Vehicle / Patent / Other
   - Estimated value in ETH: enter 0.5 or higher
   - Organization name: verify it is pre-filled, then edit if needed
   - Supporting document note: "e.g. registration number 1234"
4. Click Submit and verify the asset is added to the list.
5. If assets exist, click the details icon for one asset.
6. Verify the drawer shows:
   - Asset full details
   - Linked loan info
   - On-chain token ID
   - Blockchain TX hash with external link icon
7. If the asset status is COLLATERAL and the linked loan is REPAID, verify the "Release guarantee" button appears.
8. If it appears, click it and confirm the guarantee is released.
9. If no assets exist, verify the empty state text and button appear: "No assets provided yet" and "Add your first asset." 

Scenario 6: Governor Dashboard
1. Log in as a Governor user.
2. Go to /governor/dashboard.
3. Confirm the top stat cards show:
   - Total users
   - Pending approvals (amber if > 0)
   - Active loans
   - Platform TVL
4. In the "Pending registrations" preview table, confirm there are up to 3 entries.
5. Click "View all" to go to /governor/registrations.
6. Verify each pending row has Approve and Reject inline buttons.
7. In the "Recent audit events" list, confirm each row shows:
   - Event badge
   - Wallet address in monospace format
   - TX hash with external link
   - Timestamp

Scenario 7: Governor Registration Requests
1. At /governor/registrations, verify the header shows "Registration requests" and a pending count badge.
2. In the Pending tab, verify the table columns include:
   - User details with avatar initials and wallet
   - Role badge
   - Submitted time as relative text (e.g. "2h ago")
   - Actions
3. Click "View details" on a pending request.
4. In the details dialog, verify full profile data appears.
5. Click Approve and follow the blockchain transaction flow.
6. Confirm the dialog shows TxFeedbackComponent messages and moves the row to the Approved tab.
7. Click Reject and enter a rejection reason if prompted.
8. Confirm the request moves to the Rejected tab without on-chain transaction.
9. Switch to Approved and Rejected tabs and verify entries appear there.

Scenario 8: Governor Access Rights
1. Go to /governor/access.
2. Confirm the table columns show:
   - User with avatar and wallet
   - Role badge
   - Status badge
   - Approved date
   - Actions
3. In the search field, type a user name or wallet address to filter results.
4. Click the revoke icon button for a user.
5. Confirm the revoke dialog appears and follow the TxFeedback flow.
6. After success, verify the user row is removed from the table.

Scenario 9: Governor Platform Parameters
1. Go to /governor/parameters.
2. Confirm the page header is "Platform parameters." 
3. Verify the form fields exist:
   - Max loan amount (ETH)
   - Collateral ratio slider
   - Min interest rate (%) and Max interest rate (%)
   - Loan durations editable chips
4. Move the collateral ratio slider and verify the live text updates: "Loan = X% of collateral value."
5. Add a new chip duration such as 45 days and remove one existing duration.
6. Click "Save parameters." 
7. Verify a success message appears and the Last updated date is refreshed.

Scenario 10: Governor Audit Log
1. Go to /governor/audit.
2. Confirm the header says "Audit log" and there is a date range picker.
3. Use the event type dropdown to select each available event type in turn and verify the table updates.
4. Set a start and end date in the date range picker and click Filter.
5. Confirm the table shows results for the selected date range.
6. Verify the audit table columns include:
   - Event type badge
   - Actor wallet (monospace)
   - Details
   - TX hash with external link
   - Timestamp

Additional testing tips:
- If a page shows blank data, verify whether that is an actual empty-state scenario or an error message.
- When a wallet transaction is requested, check for the MetaMask or wallet confirmation popup and approve it.
- Capture screenshots for any broken page, missing button, or incorrect text.
- Write down the exact page name, the step where it failed, and the visible message.

End of instructions.
